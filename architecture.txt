Helm is a package manager for k8s. We can reuse the k8s manifest by specifying the value somewhere.
Track deployments

Charts: metadata about the charts
Directory structure of Helm
1. Charts.yaml contains metadata about the charts
2. template.yaml contains actual deployment files
3. values.yaml contains values of deployment files
4. README.md information about the Charts
5. _helpers.tpl define the functions
6. test-connections.yaml before doing the actualy deployment run some function. Then this can be used.
7. notes.txt once the chart is deployed as output you can see this there.
8. .helmigore files dont want to include in helms
9. charts directory store dependancies
